Dash Quest, by Tiny Titan

Dash quest is an infinite runner Android game. It has some RPG aspects. I installed the game on my tab one week ago. In only one day, I got stuck into it. Dash quest is a really addicting piece of art. So what is the secret? what are the things that make this game that addicting? One week passed and there was not only a day I did not play the game. The first impression you get from it, after installation is that it looks simple, a bit too simple. You would even think it was made by a student as a test or some similar purpose. The gameplay is so simple that it made me think of those tap games or click games where you only have to tap eternally on the screen or click with the mouse for PC gamers. But there is more behind this simplicity. By the time you get addicted, it will become so complicated that you have to read and learn the beastiary and the help pages to evolve in the game. First, let's look at the art side of the game. As I said earlier, it looks simple. The graphics are made of pixel art. Some people think that pixel art is outdated and ugly. But for others, it is special and beautiful. Pixel arts are not easy to realize. It takes talents, time and dedication. It can takes weeks of work to only get 5 sec of really good animation. Nowadays, for having the courage to dive into the world of pixel art, you have to be a very patient person. In the game we are talking about, the art is colorful, lively and appealing. The animation is fluid and crispy. The items are so well designed. The main character has a different look for each armor it wears. The game art has so much details that you would ask why is the game free. What about the audio? As far as I know, there is only one soundtrack looping endlessly as background music. That is not really a problem as it is mixed with the sound effects. The music drags you so deep into the action of the game you would think you are a professional gamer trying to win the world championship. Let's now talk more about the gameplay. Dash quest is a top down infinite runner game. The main character runs by itself. The player has to make him attack, block, use items and spells. The left side of the screen is for blocking enemies' attacks. The block action has a cooldown of about 2 seconds. The right side of the screen is for the attack. If the main character has a sword, tapping on the right side of the screen would make him use it. There is another action that the main character can do. Tap and hold on the right screen charge up a spell and release a lightning beam. This action does not need mana. The items you can hold during the game are limited to only three slots. And same thing with the spells, you only have a maximum of three spells. But you can do more! The game offers the possibility to own a pet. Each pet has its own skill. Before choosing your pet, usually after few hours into the game, you have to know what you want for you main character. Is it intelligence? Is it shire power? Do you want your main character to become a ninja or a berserker? It is only after answering those questions that you know which pet is best for you. bla

Construct 2, another software I would like to master

I started learning construct 2 a year ago. It is an incredibly easy game engine for Microsoft Windows. I use the free version for now as I am not yet at a professional level. Construct 2 is one of the easiest game engines out there. Usually compared to YoYo Game's Game Maker and Stencyl, it is cross platform and very affordable. When it comes to game engines, what people think of first is programming language. Which programming language is used in construct 2? Practically, there is none. Events are the main elements the build up the logic of the game. Of course, for advanced users, it is possible to use JavaScript for external plug-in. But even in that case, it would not be done inside the software itself. The UI of construct 2 looks like a graphics software. Sometime it can make you think of Autodesk's applications because of the disposition of the layer panel, the project panel, properties. It is also a bit similar to Adobe's Adobe Flash, but without the timeline. A project has two main folders that are the Layouts and the Event sheets. A layout is a window where the levels are designed. An event sheet is where the game logic is written. Events are some sort of premade functions that can be used easily to create an action. Most of events are similar to if statements in traditional programming languages. For example, if an object in the scene is touched by another object, it is possible to make that action trigger another action such as a sound effect or an explosion animation. Another good functionality of construct 2 is the easy to use monetization system. AdMob is included into the software and can be used for each game by just few clicks. Though it is not possible to export a game that has AdMob with the free version of the software, it is a good way to understand how monetization works and how easy it is to add interstitial and banner ads in a game. I choose construct because of the possibility to quickly export mobile games. With the free version, the HTML5 file exported has to be converted to a zip file and then uploaded to an Adobe Phonegap account in order to convert it to an apk file (apk is the extension for android installation files). The free version is already powerful, but if one wants to go professional and has the intention to make money with it, it is advised to get the pro version, for unlimited use of layers (there are only 4 usable layers in the free version), unlimited effects, and to be able to use the entire exportation mode.
